# Troubleshooting Guide

This guide covers common issues you might encounter while running AI Dev OS and their solutions.

---

## 1. Context Window Exhaustion

**Symptom:**
> "Warning: Context used 95%. Approaching 100k token limit."
> "Anthropic API Error: Maximum context length exceeded."

**Cause:**
The Subagent has read too many large files or the conversation history has grown too long.

**Solution:**
The `ContextManager` is designed to auto-summarize at 90%, but if a single command dumps a massive log file (e.g., 200k lines from a crash), it bypasses the threshold.
*   Ensure your agents use `grep` or `tail` rather than `cat` for reading large files.
*   Check your `AGENTS.md` and ensure `context_critical: 90` is correctly configured.

---

## 2. Docker Sandbox Creation Fails

**Symptom:**
> "DockerSandboxInitializationError: Connection refused. Is Docker running?"

**Cause:**
The orchestrator cannot communicate with the Docker daemon.

**Solution:**
1.  Ensure Docker Desktop or the Docker Engine is running on your host machine.
2.  On Windows/WSL2, ensure Docker integration with your WSL distribution is enabled in Docker Settings -> Resources -> WSL Integration.
3.  Check if your user has permission to use Docker (`sudo usermod -aG docker $USER`).

---

## 3. Unsloth OutOfMemory (OOM) Errors

**Symptom:**
> "CUDA OutOfMemory Error during training step 45."

**Cause:**
Model fine-tuning is attempting to use more VRAM than available on your GPU hardware or your cloud allocation.

**Solution:**
1.  Ensure you have at least 16GB of VRAM for 7B/8B parameter models, even with Unsloth limits.
2.  Lower your batch size in the agent's training script generated by open-swe.
3.  Ensure 4-bit quantization is enabled in the `AGENTS.md` Unsloth rules block.
4.  If running locally, switch your execution environment to Modal for cloud GPU resources: `python scripts/setup-sandboxes.py --provider modal`.

---

## 4. Claude HUD Missing Output

**Symptom:**
The terminal UI shows garbled text, or progress bars overwrite themselves violently.

**Cause:**
The terminal does not support advanced ANSI escape codes or is too narrow.

**Solution:**
1.  Ensure your terminal width is at least 120 columns.
2.  Use a modern terminal emulator (Windows Terminal, iTerm2, Kitty).
3.  Disable Claude HUD entirely by passing `--no-hud` to the cli, which falls back to standard line-based `logging` output.

---

## 5. API Key Missing

**Symptom:**
> "CRITICAL SECURITY ERROR: ANTHROPIC_API_KEY environment variable is missing."

**Cause:**
The orchestrator cannot call Claude because the secret token is not set in the shell environment.

**Solution:**
Export your Anthropic API Key before running python:
```bash
export ANTHROPIC_API_KEY=sk-ant-api03-...
```
Or place it inside your `.env` file at the root of the project.
